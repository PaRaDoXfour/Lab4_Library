## 🛠 Інструкція з оновлення та відкату (Update & Rollback)

Цей розділ призначений для DevOps/Release інженерів і описує процедуру переходу на нову версію бібліотечної системи.

### 1. Підготовка до оновлення
Перш ніж вносити зміни у виробниче середовище, виконайте наступні кроки:

* **Створення резервної копії (Backup):** Обов’язково скопіюйте поточний стан бази даних та стабільної версії коду:
```bash
mkdir -p ./backups/$(date +%F)
cp input.txt ./backups/$(date +%F)/input.txt.bak
cp Lab4_Library.jar ./backups/$(date +%F)/Lab4_Library.jar.old
```
* **Перевірка сумісності:** Переконайтеся, що нова версія не вимагає оновлення JRE (наприклад, перехід з Java 17 на Java 21). Перевірте `RELEASE_NOTES` на предмет змін у форматі файлу `input.txt`.
* **Планування простою (Downtime):** Оскільки програма працює з файлом монопольно, під час оновлення користувачі не зможуть вносити зміни. Запланований час простою: 5-10 хвилин.

### 2. Процес оновлення
Виконуйте кроки суворо в наведеній послідовності:

1. **Зупинка активних процесів:** Знайдіть та завершіть процес Java, щоб уникнути пошкодження файлу даних:
```bash
pgrep -f Lab4_Library | xargs kill -15
```
2. **Розгортання нового коду:** Замініть старий JAR-файл новим артефактом:
```bash
cp /path/to/new/version/Lab4_Library-v2.jar ./Lab4_Library.jar
```
3. **Міграція даних:** Якщо нова версія змінює структуру `input.txt` (наприклад, додано нові поля для `EBook`), запустіть скрипт міграції або відредагуйте файл згідно з новою специфікацією.
4. **Оновлення конфігурацій:** Якщо у новій версії додалися нові змінні оточення або параметри запуску, оновіть скрипт запуску (`start.sh`).

### 3. Процедура відкату (Rollback)
Якщо після оновлення система не запускається або дані відображаються некоректно, негайно поверніться до стабільної версії:

1. **Зупиніть "биту" версію:** `pkill -f Lab4_Library`.
2. **Відновіть код:** Поверніть старий JAR-файл з теки backup.
```bash
cp ./backups/$(date +%F)/Lab4_Library.jar.old ./Lab4_Library.jar
```
3. **Відновіть дані:** Поверніть резервну копію `input.txt`.
```bash
cp ./backups/$(date +%F)/input.txt.bak ./input.txt
```
4. **Запуск:** Запустіть систему та перевірте логі на відсутність помилок.
